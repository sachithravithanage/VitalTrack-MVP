# PDF Export Feature - Fix Documentation

## Issue

The PDF export feature was generating local PDFs using the Flutter printing package instead of downloading the PDFs generated by the backend and stored in Firebase Storage.

## Solution

Modified the PDF export functionality to:

1. Call the backend API to generate the PDF
2. Get the signed download URL from Firebase Storage
3. Open the PDF URL in the default application (browser or PDF viewer)

## Changes Made

### Backend (Already Implemented)

- **`POST /api/v1/records/export/pdf`** - Generates PDF report and stores in Firebase Storage
  - Returns a signed URL valid for 7 days
  - Includes patient information and all medical records
  - Professional formatted report with headers and footers

### Frontend Changes

#### 1. **records.dart** - Updated PDF export flow

```dart
// OLD: Used local Printing package
await Printing.layoutPdf(onLayout: (format) => pdf.save());

// NEW: Calls backend API and downloads PDF
final String pdfUrl = await app.exportRecordsPdf(filter: _filter);
if (await canLaunchUrl(uri)) {
  await launchUrl(uri, mode: LaunchMode.externalApplication);
  // Shows success message
}
```

**Changes:**

- Removed `pdf` and `printing` package imports
- Added `url_launcher` import
- Replaced local PDF generation with backend API call
- Opens PDF URL in external application (default browser/PDF viewer)
- Shows user-friendly messages on success/failure

#### 2. **pubspec.yaml** - Added dependency

```yaml
dependencies:
  url_launcher: ^6.2.0
```

#### 3. **localization.dart** - Added translation strings

- `export_failed` - "Failed to export PDF"
- `pdf_downloaded` - "PDF downloaded successfully"
- `cannot_open_pdf` - "Cannot open PDF. Please try again."

## How It Works

### User Flow:

1. User navigates to Records screen
2. Selects records using timeline filter (Last 24h, 3 days, 7 days)
3. Clicks "Export PDF" button
4. Backend API:
   - Generates professional PDF report
   - Stores in Firebase Storage
   - Returns signed download URL (valid 7 days)
5. Frontend:
   - Opens PDF URL in default application
   - On Web: PDF downloads
   - On Mobile: PDF opens in browser or PDF viewer
6. User sees success message

### API Response Format:

```json
{
  "success": true,
  "data": {
    "pdf": {
      "fileName": "medical-records-<patientId>-<timestamp>.pdf",
      "filePath": "exports/medical-records-<patientId>-<timestamp>.pdf",
      "url": "https://storage.googleapis.com/<bucket>/<filePath>?<signedUrlParams>",
      "size": 15234,
      "createdAt": "2026-03-19T..."
    }
  }
}
```

## PDF Report Contents

- **Header**: VitalTrack Medical Report + Generation timestamp
- **Patient Info**: Name, Email, Phone
- **Medical Records**:
  - Record number and disease type
  - Date and time
  - Temperature (°C)
  - Fluid intake/output
  - Urine color
  - Symptoms
  - Notes
- **Footer**: Disclaimer about consulting healthcare provider

## Frontend Service Layer

The service layer (state.dart) handles:

```dart
Future<String> exportRecordsPdf({
  TimelineFilter filter = TimelineFilter.last7Days,
}) async {
  // Calls backend API
  final response = await recordService.exportRecordsPdf(...);

  // Extracts URL from response
  return pdf['url'] ?? '';
}
```

## Error Handling

- **No records to export**: Shows error message
- **PDF generation fails**: Backend returns error
- **URL launch fails**: Shows "Cannot open PDF" message
- **Network error**: Shows error message with details

## Platform Behavior

### Android/iOS

- Opens PDF in default PDF viewer or browser
- May show "Save as" dialog depending on browser
- PDF displayed in native viewer

### Web

- Automatically downloads PDF to Downloads folder
- Respects browser download settings
- PDF shown in browser or downloaded depending on configuration

### Windows/macOS

- Opens in default PDF application
- Better for desktop workflows

## Security

The PDF export feature includes:

- **Authentication**: Only authenticated users can export
- **Authorization**: Patients can only export their own records
- **Signed URLs**: Firebase Storage URLs are signed and expire after 7 days
- **Data Privacy**: PDFs stored in Google Cloud Storage with proper permissions
